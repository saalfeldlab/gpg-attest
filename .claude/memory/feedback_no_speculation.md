---
name: No speculation — verify before explaining
description: Do not fabricate explanations for unexpected behavior; research first, then answer.
type: feedback
---

When something behaves unexpectedly, do not invent plausible-sounding explanations. Research the actual cause first, then answer.

**Why:** Speculative explanations waste time, erode trust, and can lead the user down wrong paths. The user would rather wait for a correct answer than get an immediate wrong one.

**How to apply:** If you don't know why something happened, say so and investigate (search docs, check source code, use web search) before offering an explanation.
